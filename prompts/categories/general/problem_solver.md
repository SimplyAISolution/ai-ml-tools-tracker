# Systematic Problem Solver

## Description
Breaks down complex problems into manageable components and provides structured solutions with multiple approaches and implementation strategies.

## Prompt
```
I need help solving a complex problem. Please analyze it systematically and provide comprehensive solutions.

**Problem Statement**: {problem_description}
**Context/Background**: {relevant_context}
**Constraints**: {limitations_or_constraints}
**Success Criteria**: {how_to_measure_success}
**Timeline**: {available_timeframe}

Please provide a structured analysis using the following framework:

## Problem Analysis
### 1. Problem Breakdown
- **Core Issue**: [Main problem to solve]
- **Sub-problems**: [Break into smaller components]
- **Root Causes**: [Underlying causes, not just symptoms]
- **Stakeholders**: [Who is affected/involved]

### 2. Constraint Analysis
- **Hard Constraints**: [Unchangeable limitations]
- **Soft Constraints**: [Flexible limitations]
- **Resource Requirements**: [Time, money, people, tools needed]

## Solution Framework
### 3. Approach Options
- **Option A**: [Conservative/low-risk approach]
  - Pros: [Benefits and advantages]
  - Cons: [Drawbacks and risks]
  - Feasibility: [Implementation difficulty]

- **Option B**: [Balanced/moderate approach]
  - Pros: [Benefits and advantages]
  - Cons: [Drawbacks and risks]
  - Feasibility: [Implementation difficulty]

- **Option C**: [Innovative/high-impact approach]
  - Pros: [Benefits and advantages]
  - Cons: [Drawbacks and risks]
  - Feasibility: [Implementation difficulty]

### 4. Recommended Solution
- **Primary Approach**: [Best option with reasoning]
- **Implementation Steps**: [Detailed action plan]
- **Risk Mitigation**: [How to handle potential issues]
- **Success Metrics**: [How to measure progress]

### 5. Contingency Planning
- **Alternative Paths**: [Backup plans if primary fails]
- **Early Warning Signs**: [Indicators of potential problems]
- **Pivot Strategies**: [When and how to change course]

Provide specific, actionable advice with clear next steps.
```

## Example Usage
```
Input:
- Problem: Low employee engagement and high turnover in remote team
- Context: 50-person tech company, fully remote since 2020
- Constraints: Limited budget, can't increase salaries significantly
- Success Criteria: Reduce turnover by 30%, improve engagement scores
- Timeline: 6 months

Output:
## Problem Analysis
### 1. Problem Breakdown
- **Core Issue**: Disconnect between remote employees and company culture
- **Sub-problems**: 
  - Lack of meaningful peer connections
  - Limited career development visibility
  - Insufficient recognition and feedback
  - Work-life boundary issues
- **Root Causes**: 
  - Inadequate onboarding for remote culture
  - Missing informal interaction opportunities
  - Unclear growth pathways
  - Overemphasis on productivity metrics
- **Stakeholders**: Employees, managers, HR, leadership team

### 2. Constraint Analysis
- **Hard Constraints**: $10K budget, 6-month timeline, remote-only environment
- **Soft Constraints**: Manager time availability, employee participation willingness
- **Resource Requirements**: HR coordination, technology tools, manager training

## Solution Framework
### 3. Approach Options
- **Option A: Culture-First Approach**
  - Pros: Addresses root engagement issues, sustainable long-term
  - Cons: Takes time to show results, requires consistent commitment
  - Feasibility: High, leverages existing team structure

- **Option B: Systems & Process Improvement**
  - Pros: Measurable improvements, addresses operational pain points
  - Cons: May not solve deeper engagement issues
  - Feasibility: Medium, requires tool investments

- **Option C: Comprehensive Overhaul**
  - Pros: Addresses all issues simultaneously, maximum impact
  - Cons: Resource-intensive, risk of overwhelming team
  - Feasibility: Low given constraints

### 4. Recommended Solution: Hybrid Culture-Systems Approach
**Implementation Steps**:
1. Month 1: Launch virtual coffee chats and interest-based Slack channels
2. Month 2: Implement bi-weekly one-on-ones with career development focus
3. Month 3: Create peer recognition system and monthly team showcases
4. Month 4: Establish mentorship program and skills sharing sessions
5. Month 5-6: Measure progress and refine successful initiatives

**Risk Mitigation**:
- Survey employees monthly for feedback and adjustment
- Train managers on remote engagement techniques
- Create backup communication channels if primary tools fail

**Success Metrics**:
- Monthly engagement surveys (target: 15% improvement)
- Turnover tracking (target: 30% reduction)
- Participation rates in new programs (target: 70%+)
```

## Tips
- Be specific about timelines and resource requirements
- Consider both short-term wins and long-term solutions
- Address emotional and practical aspects of problems
- Include measurement and feedback mechanisms
- Plan for multiple scenarios and contingencies
- Focus on actionable steps rather than theoretical solutions
